# Level 4: Specialization Tracks

> **Duration:** 1-2 hours per track
> **Prerequisites:** Level 1 required, additional levels recommended per track

## Overview

Level 4 provides role-specific training that focuses on what each stakeholder needs to know about AI governance. Each track covers the same system from a different perspective.

## Available Tracks

| Track | Duration | Target Audience | Prerequisites |
|-------|----------|-----------------|---------------|
| [4.1 Developer Track](./01-developer-track.md) | 1.5-2 hrs | Software Engineers, ML Engineers | L1, L2, L3.3 |
| [4.2 Product Manager Track](./02-product-manager-track.md) | 1-1.5 hrs | Product Managers, Product Owners | L1 |
| [4.3 CISO/Security Track](./03-ciso-security-track.md) | 1-1.5 hrs | CISOs, Security Engineers | L1, L3.2 |
| [4.4 Legal/Compliance Track](./04-legal-compliance-track.md) | 1.5-2 hrs | Legal, Compliance Officers | L1, L3.2 |

## Track Selection Guide

```
What is your primary role?
│
├─► Building AI systems
│   └─► Developer Track (4.1)
│
├─► Defining AI products
│   └─► Product Manager Track (4.2)
│
├─► Securing the organization
│   └─► CISO/Security Track (4.3)
│
└─► Ensuring compliance
    └─► Legal/Compliance Track (4.4)
```

## Cross-Functional Understanding

While each track focuses on role-specific content, all stakeholders benefit from understanding other perspectives:

| If you are... | Also consider... |
|---------------|------------------|
| Developer | Sections from 4.4 on regulatory requirements |
| Product Manager | Sections from 4.1 on technical constraints |
| CISO | Sections from 4.4 on legal implications |
| Legal/Compliance | Sections from 4.3 on security controls |
